Основные команды Docker

Работа с контейнерами

  • docker run — создаёт и запускает контейнер из образа. Принимает параметры, которые определяют настройки контейнера.
  • docker ps — выводит список активных контейнеров в системе. Флаг -a показывает все контейнеры, включая остановленные.
  • docker stop — останавливает работающий контейнер.
  • docker restart — перезапускает контейнер.
  • docker rm — удаляет один или несколько контейнеров, которые уже остановлены.

Работа с образами

  • docker pull — загружает образы из удалённого репозитория Docker Hub. Можно указать версию образа
  • docker images — выводит список загруженных образов.
  • docker rmi — удаляет образы с локального хоста.
  • docker search — ищет определённый образ в Docker Hub.

Работа с сетями и томами

  • docker network create — создаёт новую сеть.
  • docker network ls — выводит список всех сетей.
  • docker network connect — подключает контейнер к сети.
  • docker network disconnect — отключает контейнер от сети.
  • docker volume create — создаёт новый том для сохранения данных контейнера.
  • docker volume ls — выводит список всех томов.
  • docker volume rm — удаляет том, если он не используется контейнерами.

Полезные команды

  • docker system prune — очищает систему, удаляя неиспользуемые образы, контейнеры и сети.
  • docker logs — выводит логи контейнера.
  • docker exec — открывает интерактивный терминал внутри запущенного контейнера.
